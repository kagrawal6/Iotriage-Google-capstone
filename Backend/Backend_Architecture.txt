IoTriage Backend Flowchart (Chronological)

[Frontend React UI: User Action]
        |
        | User uploads Nmap scan JSON
        | POST /api/upload
        v
[Routes Layer: scanRoutes.js]
        |
        | Maps URLs to controller methods:
        | - POST /api/upload   -> scanController.uploadScan()
        | - GET /api/devices   -> scanController.getDevices()
        | - GET /api/results   -> cveController.getVulnerabilities()
        v
[Scan Controller: scanController.js]
        |
        | Methods:
        | - uploadScan(req, res)
        |     -> clears previous data (deviceModel.clearDevices, vulnerabilityModel.clearVulnerabilities)
        |     -> adds Device objects (deviceModel.addDevices)
        |     -> iterates each Device:
        |           - getCPEs() from device
        |           - fetch CVEs via nvdService.fetchCVEs(cpe)
        |           - store CVEs in VulnerabilityModel.addVulnerability()
        |     -> call llmService.createMitigationSteps()
        |     -> respond with JSON { devices, vulnerabilities, mitigations }
        | - getDevices(req, res)
        |     -> return all Device objects in memory
        v
[Device Model: deviceModel.js]
        |
        | Classes & Methods:
        | - Device
        |     - constructor(raw JSON)
        |     - getCPEs() -> returns array of CPE strings
        | - DeviceModel
        |     - addDevice(deviceData)
        |     - addDevices(array)
        |     - getAllDevices()
        |     - clearDevices()
        v
[NVD Service: nvdService.js]
        |
        | Methods:
        | - fetchCVEs(cpe) -> returns array of CVE objects
        v
[Vulnerability Model: vulnerabilityModel.js]
        |
        | Classes & Methods:
        | - Vulnerability
        |     - constructor({cveId, description, severity, deviceIp})
        | - VulnerabilityModel
        |     - addVulnerability(vulnData)
        |     - getAllVulnerabilities()
        |     - clearVulnerabilities()
        v
[LLM Service: llmService.js]
        |
        | Methods:
        | - summarizeCVE(cveData) -> plain-English explanation
        | - createMitigationSteps(vulnerabilities) -> actionable steps
        v
[Frontend React UI: Display Results]
        |
        | Receives JSON response:
        | {
        |   "devices": [...],
        |   "vulnerabilities": [...],
        |   "mitigations": [...]
        | }
        | Renders cards, tables, and mitigation steps



Chronological Steps (Step-by-Step)

1. User uploads JSON scan → frontend sends POST /api/upload.

2. Routes layer maps /api/upload → scanController.uploadScan().

3. scanController.uploadScan:
   - Clears all previous devices and vulnerabilities.
   - Adds new Device objects using deviceModel.addDevices().
   - Iterates through devices:
       * Calls device.getCPEs() to extract CPE strings.
       * Calls nvdService.fetchCVEs(cpe) to get CVEs.
       * Stores each CVE in vulnerabilityModel.addVulnerability().
   - Calls llmService.createMitigationSteps() to produce human-readable guidance.

4. scanController.uploadScan responds to frontend with JSON:
   {
     "devices": [...],
     "vulnerabilities": [...],
     "mitigations": [...]
   }

5. Frontend displays all data:
   - Devices → table/cards
   - Vulnerabilities → table/cards
   - Mitigation steps → instructions for user

6. User can later call:
   - GET /api/devices → retrieves all Device objects in memory
   - GET /api/results → retrieves all Vulnerabilities in memory
