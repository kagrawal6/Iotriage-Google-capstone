Backend Architecture:


IoTriage Backend Flowchart (Chronological with Frontend at Top & Bottom)
[Frontend React UI: User Action]
        |
        | User clicks "Scan"
        | POST /api/scan
        v
[Routes Layer: scanRoutes.js]
        |
        | Maps URLs → controller methods:
        | - POST /api/scan       -> scanController.runScan()
        | - GET  /api/devices    -> scanController.getDevices()
        | - GET  /api/results    -> cveController.getVulnerabilities() 
        v
[Scan Controller: scanController.js]
        |
        | Methods:
        | - runScan()             -> triggers Python + Nmap scanner
        |                           -> converts output to device objects
        |                           -> queries NVD API for each device CPE
        |                           -> stores devices + vulnerabilities
        |                           -> calls LLM to generate mitigation steps
        | - parseScanOutput()     -> helper: converts scanner JSON → device objects
        | - getDevices()          -> retrieves stored devices
        v
[Python + Nmap Scanner]
        |
        | Methods:
        | - scanNetwork()         -> discovers IPs, ports, services, OS
        | - saveJSON()            -> outputs JSON for backend
        v
[Device Model: deviceModel.js]
        |
        | Methods:
        | - addDevice(device)     -> store a device
        | - getAllDevices()       -> retrieve all devices
        | - clearDevices()        -> remove all devices
        v
[NVD API]
        |
        | Methods:
        | - fetchCVEs(cpe)        -> return potential vulnerabilities for device
        v
[Vulnerability Model: vulnerabilityModel.js]
        |
        | Methods:
        | - addVulnerability(vuln) -> store a CVE
        | - getAllVulnerabilities() -> retrieve CVEs
        | - clearVulnerabilities()  -> remove CVEs
        v
[LLM Integration]
        |
        | Methods:
        | - summarizeCVE(cveData) -> plain-English explanation
        | - createMitigationSteps() -> actionable advice for user
        v
[Frontend React UI: Display Results]
        |
        | Receives JSON response:
        | {
        |   "devices": [...],
        |   "vulnerabilities": [...],
        |   "mitigations": [...]
        | }
        | Renders cards, tables, and mitigation steps


Chronological Steps (Step-by-Step)

1. User clicks "Scan" → frontend sends POST /api/scan.
2. Routes layer (scanRoutes.js) maps /api/scan → scanController.runScan()
3. scanController.runScan():
       - Runs Python + Nmap scanner → scanNetwork() → JSON output.
       - parseScanOutput() converts JSON → device objects.
       - Stores devices → deviceModel.addDevice().
4. For each device:
       - Convert device → CPE string.
       - Query NVD API → fetch potential CVEs.
       - Store CVEs → vulnerabilityModel.addVulnerability().
5. LLM Integration:
       - summarizeCVE() → produces plain-English explanation.
       - createMitigationSteps() → actionable steps for user.
6. scanController.runScan() returns a single JSON response with devices, vulnerabilities, and mitigation steps.
7. Frontend React UI receives JSON → renders cards, tables, and explanations for the user.
