Backend Architecture:

[Frontend React UI]
        |
        | -- 1) Upload Nmap scan JSON -->
        | POST /api/upload
        v
[Routes/ScanRoutes.js]
        |
        | Maps URL → scanController.uploadScan()
        v
[Controllers/ScanController.js]
        |
        | - Converts raw JSON → Device objects
        | - For each device, extracts CPEs (device.getCPEs())
        | - For each CPE, fetch CVEs via nvdService.fetchCVEs(cpe)
        | - Create Vulnerability objects
        | - Returns JSON response:
        |   {
        |     devices: [...],
        |     vulnerabilities: [...]
        |   }
        v
[Services/NvdService.js]
        |
        | Placeholder logic:
        | - Receives CPE
        | - Returns CVE array (id, description, severity)
        v
[Frontend React UI]
        |
        | Receives JSON response:
        | - Renders devices table/cards
        | - Renders vulnerabilities table/cards
        |
        | -- 2) Chat/AI interaction -->
        | POST /api/chat
        v
[Routes/ChatRoutes.js]
        |
        | Maps URL → chatController.sendMessage()
        v
[Controllers/ChatController.js]
        |
        | - Receives chatHistory and new user message
        | - Calls llmService.sendChatToLLM(chatHistory, message)
        | - Returns AI response JSON
        v
[Services/LlmService.js]
        |
        | Placeholder LLM integration:
        | - Receives chat context & new message
        | - Returns AI-generated response string
        v
[Frontend React UI]
        |
        | Displays AI response to user in chat interface




Chronological Steps (Step-by-Step)

1. User uploads Nmap scan JSON from the frontend.

2. Frontend sends POST /api/upload with the scan JSON.

3. ScanRoutes.js maps the request to scanController.uploadScan().

4. scanController.uploadScan():
   - Converts raw JSON → Device objects.
   - For each device:
       - Calls device.getCPEs() to extract CPE strings.
       - Queries NVD via nvdService.fetchCVEs(cpe) to get CVEs.
       - Creates Vulnerability objects for each CVE.
   - Returns JSON:
     {
       "devices": [...],
       "vulnerabilities": [...]
     }

5. Frontend receives JSON and renders:
   - Devices table/cards.
   - Vulnerabilities table/cards.

6. User sends a chat message (e.g., "What should I do about CVE-2023-1234?").

7. Frontend sends POST /api/chat with:
   - chatHistory (full local conversation)
   - message (new user input)

8. ChatRoutes.js maps the request to chatController.sendMessage().

9. chatController.sendMessage():
   - Calls llmService.sendChatToLLM(chatHistory, message)
   - Receives AI-generated response
   - Returns JSON:
     { "reply": "AI response" }

10. Frontend receives AI response and displays it in the chat interface.
